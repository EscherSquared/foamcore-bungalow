TCJ Foam Core Bungalow Project
Grade 9 culminating project: students build a one-storey bungalow scale model from a single sheet of foam core. Includes three designs (two easy, one medium), printable cut lists, instruction sheet, reflection page, and assessment rubric.
What's in this repo
```
website/
├── index.html              ← Main page: 3 bungalow designs with drawings & cut lists
├── documents.html          ← Download page for the 4 student/teacher PDFs
├── styles.css              ← Stylesheet (blueprint aesthetic)
└── pdfs/
    ├── TCJ_Bungalow_Instructions.pdf   (2 pages)
    ├── TCJ_Bungalow_Cutlists.pdf       (3 pages — one design per page)
    ├── TCJ_Bungalow_Reflection.pdf     (2 pages)
    └── TCJ_Bungalow_Rubric.pdf         (1 page)
```
Hosting on GitHub Pages
Create a new repository on GitHub (e.g. `foamcore-bungalow`). Make it public.
Upload the contents of the `website/` folder to the repo root. Either:
Drag and drop in the GitHub web UI, or
From a terminal:
```
     git clone https://github.com/YOUR-USERNAME/foamcore-bungalow.git
     cp -r website/* foamcore-bungalow/
     cd foamcore-bungalow
     git add . && git commit -m "Initial site"
     git push
     ```
Enable GitHub Pages: in the repo, go to Settings → Pages. Under "Source", pick Deploy from a branch, choose `main` branch and `/ (root)` folder. Save.
Wait 1–2 minutes. Your site will be live at `https://YOUR-USERNAME.github.io/foamcore-bungalow/`.
Share that URL with students. They can navigate the designs, view the drawings, and download the four PDFs from the Documents page.
Project specs
Material: one standard 50 × 76 cm foam core sheet per student (5 mm thick)
Units: metric (cm/mm)
Designs:
A — Modern Shed-Roof (easy, 6 pieces, 20 × 15 cm footprint)
B — L-Shaped (medium, 9 pieces, 20 × 20 cm L footprint)
C — Classic Pitched-Roof (easy, 7 pieces, 18 × 18 cm footprint)
Assessment: 5-criterion rubric, 20 points total, aligned to Ontario Achievement Chart (Thinking, Application, Communication)
Modifying the project
The four PDFs are generated by `build_pdfs.py` (not included in the website folder — kept in the project root). Re-running that script after editing will regenerate all four documents.
